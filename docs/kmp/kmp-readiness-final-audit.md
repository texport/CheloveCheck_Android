# Готовность KMP — финальный аудит (data-слой)

Дата: 2026-04-08. Область: [`shared/data-core`](../../shared/data-core), интеграция в `:app`, границы из [`data-core-cut-map.md`](data-core-cut-map.md).

## Критерии (план, §9.8)

| Критерий | Статус | Обоснование |
|----------|--------|---------------|
| В `commonMain` нет Android API | Выполнено | `SharedOfdSanitizer` и `Platform` (expect/actual) в `commonMain` / `iosMain` / `jvmMain`; в `commonMain` нет импортов `android.*`. |
| Сборка shared для JVM и iOS | Выполнено | В Gradle включены KMP-таргеты; локально гоняется `:shared:data-core:allTests` (аналог CI). |
| Android подключает shared явно | Выполнено | Модуль приложения зависит от `:shared:data-core`; `OfdResponseSanitizer` делегирует санитизацию в shared. |
| Платформенные хуки | Частично / дальше | `Platform` — минимальная проверка имени; далее по миграции: время, безопасное KV, HTTP-фасад (см. cut map). |
| Интеграция с iOS-приложением | Не начата | iOS-клиента нет в репозитории; заглушки `iosMain` + нативные тесты подтверждают только toolchain. |

## Вердикт по готовности

- **Готово к дальнейшей пошаговой миграции** платформенно-нейтральных парсеров и политик в `commonMain` без переделки архитектуры.
- **Это не утверждение «iOS-приложение готово»**: готовность — про границы модулей, форму кода и тесты, а не про выкатанный UI под iOS.

## Рекомендуемые шаги после аудита

1. Вводить `expect`/`actual` для времени и защищённого хранилища только когда появится первый реальный iOS-потребитель.
2. Переносить следующий чистый хелпер OFD или политику списка чеков в `commonMain` вместе с фикстурами `commonTest`.
3. При изменении публичных контрактов data поддерживать актуальность [`data-layer-contracts.md`](../architecture/data-layer-contracts.md).

## Регрессионный gate

```bash
./gradlew :shared:data-core:allTests :app:testDebugUnitTest
```
